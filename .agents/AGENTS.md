# Workspace Rules

## OKF (Obsidian Knowledge Folder) Usage
פעולה לכל צורת העבודה בפרויקט: להשתמש בצורה היעילה ביותר בצורת ארגון המידע OKF.
(Rule for all workflows in the project - use the OKF knowledge organization method in the most efficient way).
When designing or implementing features (especially AI insights and data extraction), always prefer utilizing the structured nodes, edges, and metadata (triples, topics, moods) provided by the Knowledge Graph (OKF) over raw, flat transcript analysis.

## OKF Conformity for Generated Content
- **כלל קבוע**: כל תוצר חדש שמיוצר על ידי בינה מלאכותית או על ידי המערכת (תובנות, ניתוחים, סיכומים וכו') חייב לייצר ולהחזיר מידע התואם את מבנה הנתונים ושיטת ארגון המידע של OKF (כולל חילוץ שלשות/Triples לגרף הידע).
- (Mandatory Rule: Every new generated output or insight must produce and return data that conforms to the OKF knowledge structure, including extracting triples for the Knowledge Graph).
